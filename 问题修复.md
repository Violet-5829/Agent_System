## 问题四：Workflow Graph 节点不会移动

**问题描述**：
当 Agent 在工作时，前端页面的 Workflow Graph 节点不会移动，只会一直停留在 "start" 节点上。即使 trace 事件中有 `node_entered` 信息，图中当前节点的高亮也不会跟随执行进度变化。

**问题分析**：

前端 `App.vue` 中的 `activeNodeId` 计算逻辑（第 74-81 行）：
```javascript
const activeNodeId = computed(() => {
  if (replayingTrace.value) return replayNodeId.value;
  const trace = lastRun.value?.trace || [];
  for (let index = trace.length - 1; index >= 0; index -= 1) {
    if (trace[index].payload?.node_id) return trace[index].payload.node_id;
  }
  return "";
});
```

后端发出的 trace 事件（如 `peer_handoff/workflow.py` 第 149 行）：
```python
push(_event("node_entered", f"执行: {worker.name}", f"流水线阶段: {worker.name}"))
```

后端 `TraceEvent` 的 payload 结构中 **没有 `node_id` 字段**，只有 `detail` 等信息。前端期望从 `trace[index].payload.node_id` 获取当前节点 ID，但后端根本没有这个字段。

**修复位置**：
1. `backend/app/workflows/*/workflow.py` - 各工作流文件中的 `_event()` 调用，需要在 `payload` 中添加 `node_id` 字段
2. `frontend/src/App.vue` 第 74-81 行 - `activeNodeId` 计算逻辑

---

## 问题分析与解决方案

### 根因

前端 `App.vue` 的 `activeNodeId` 计算属性从 `trace[index].payload.node_id` 获取当前高亮节点，但后端所有工作流发出的 `_event("node_entered", ...)` 调用中 **未传递 `node_id` 字段**，`payload` 始终为空对象 `{}`。因此 `activeNodeId` 永远返回空字符串，Graph 节点的 active 高亮状态不会更新。

### 节点 ID 映射规则

每个工作流通过 LangGraph `StateGraph.add_node(id, ...)` 注册节点，前端通过 `workflow_graph_from_compiled()` 提取节点 ID。后端 trace 事件必须使用与 Graph 构建相同的节点 ID：

| 工作流 | 逻辑节点 | Agent 节点 | 终节点 |
|--------|---------|-----------|--------|
| single_agent_chat | `single_agent` | `agent.id` | `finalize` |
| router_specialists | `router` | `agent.id` | `finalize` |
| planner_executor | `planner_core`, `planner_validator`, `task_dispatcher` | `agent.id` | `synthesizer` |
| supervisor_dynamic | `supervisor_intake`, `delegation_policy`, `supervisor_review` | `agent.id` | `finalize` |
| peer_handoff | `first_owner_router`, `handoff_decision` | `peer_exec_{agent.id}` | `finalize` |

### 修复内容

**后端（5 个文件）**：每个 `_event("node_entered", ...)` 调用增加 `{"node_id": "<对应节点ID>"}` 作为 payload 第四个参数。同时为 `run_started` 和 `run_finished` 事件添加 `node_id: "start"` / `"end"`。

**前端（App.vue）**：`activeNodeId` 计算增加 `next_node_id` 作为备用字段，默认回退值从 `""` 改为 `"start"`，确保初始状态显示起始节点。

**修改文件清单**：
- `backend/app/workflows/single_agent_chat/workflow.py`
- `backend/app/workflows/router_specialists/workflow.py`
- `backend/app/workflows/planner_executor/workflow.py`
- `backend/app/workflows/supervisor_dynamic/workflow.py`
- `backend/app/workflows/peer_handoff/workflow.py`
- `frontend/src/App.vue`
